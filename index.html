HW #1 VCS (Git, Github, gh-pages, FTP)

  • git config --global user.name “name” - встановити name користувача
  • git config --global user.email “email@google.com” - встановити email користувача
  • git init - створити новий репозиторій
  • git status - показує стан файлів в робочих папках
  • git add "file"or . - додати всі змінені файли (до stage)
  • git commit -m ‘’ "file" - створити новий коміт
  • git commit -a -m ‘’ - додати файли до stage і створити новий коміт
  • git commit --amend - оновити попередній коміт
  • git log - показати історію комітів
  • git checkout - повернутись до останнього коміту
  • git checkout "hash" - переключити репозиторій до вибраного коміту
  • git branch - показати всі гілки
  • git branch "name" - створити гілку "name"
  • git checkout "name" - переключитися на гілку "name"
  • git push - завантажити дані локального репозиторія на віддалений
  • git pull - отримати дані віддаленого репозиторія
  • git merge - з'єднати дві гілки в одну
  • git clone - клонує віддалений репозиторій
  • git revent - створення "антикоміту" для вказаного коміту(відміняє його зміни)
  • git reset - скасування змін
  • git rebase - дозволяє об'єднати зміни, зроблені в одній гілці, з іншою гілкою
  • git cherry-pick - дозволяє перенести конкретний коміт з одної гілки в іншу, без використання git merge
  • git stash - може помістити поточні зміни, які виконані в файлах, в окреме сховище, і повернути файли до вихідного стану

    • .gitignore - призначений для вказування в ньому файлів и папок, які необхідно приховати від системи контролю версій git.
